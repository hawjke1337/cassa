Required pattern (per CONTEXT.md §"Фикс 22 null-ошибок"):
```typescript
const assignments = await db.motivationAssignment.findMany({...})
const withUser = assignments.filter((a) => a.user !== null)
return withUser.map((a) => ({
  userId: a.user!.id,           // safe after filter (TS narrows)
  userName: `${a.user!.firstName} ${a.user!.lastName}`,
  // ...
}))
```

Note: after `.filter((a) => a.user !== null)` TypeScript 5+ narrows `a.user` to non-null via control flow analysis. If TS fails to narrow (older TS target), use explicit `filter<T>((a): a is T & { user: NonNullable<typeof a.user> } => a.user !== null)`.
