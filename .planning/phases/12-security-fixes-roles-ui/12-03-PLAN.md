Create new file `src/actions/roles.ts`:

```typescript
"use server"

import { auth } from "@/lib/auth"

import { revalidatePath } from "next/cache"
```

Functions to create:

1. `getRolesWithPermissions()` -- Returns all roles with their permissions for the roles table.
   - `requirePermission("settings.roles")`
   - `db.role.findMany({ include: { permissions: { include: { permission: true } }, _count: { select: { users: true } } }, orderBy: { name: "asc" } })`
   - Returns: `{ id, name, description, isSystem, permissionCodes: string[], userCount: number }[]`

2. `getRoleById(roleId: string)` -- Get single role with all permissions for editing.
   - `requirePermission("settings.roles")`
   - `db.role.findUnique({ where: { id: roleId }, include: { permissions: { include: { permission: true } } } })`
   - Returns: `{ id, name, description, isSystem, permissionCodes: string[] }`

3. `createRole(data: { name: string; description?: string; permissionCodes: string[] })` -- Create custom role.
   - `requirePermission("settings.roles")`
   - Rate limit check: `checkWriteRateLimit(session.user.id, "settings.roles")` + `recordWriteAttempt`
   - Validate: name length 1-100 chars, name unique (catch Prisma unique error)
   - Find Permission records by codes: `db.permission.findMany({ where: { code: { in: permissionCodes } } })`
   - Create Role + RolePermission entries in transaction:
     ```typescript
     await db.$transaction(async (tx) => {
       const role = await tx.role.create({
         data: {
           name: data.name.trim(),
           description: data.description?.trim() || null,
           isSystem: false,
         },
       })
       if (permissions.length > 0) {
         await tx.rolePermission.createMany({
           data: permissions.map((p) => ({ roleId: role.id, permissionId: p.id })),
         })
       }
       await createAuditEntry({
         action: "CREATE",
         entity: "Role",
         entityId: role.id,
         userId: session.user.id,
         changes: {
           name: { old: null, new: data.name },
           permissions: { old: null, new: data.permissionCodes },
         },
         tx,
       })
       return role
     })
     ```
   - `revalidatePath("/settings/roles")`

4. `updateRole(roleId: string, data: { name: string; description?: string; permissionCodes: string[] })` -- Update existing role.
   - `requirePermission("settings.roles")`
   - Rate limit check
   - Fetch existing role: `db.role.findUnique({ where: { id: roleId }, include: { permissions: { include: { permission: true } } } })`
   - If `role.isSystem` -- throw: `"Системные роли нельзя редактировать"`
   - In transaction: update role name/description, delete all existing RolePermission for this role, create new ones from permissionCodes
   - Audit entry with changes: `{ name: { old: existing.name, new: data.name }, permissions: { old: oldCodes, new: data.permissionCodes } }`
   - `revalidatePath("/settings/roles")`

5. `deleteRole(roleId: string)` -- Delete role (cascade deletes RolePermission + UserRole).
   - `requirePermission("settings.roles")`
   - Rate limit check
   - Fetch role, check isSystem -- throw if system role
   - Check if any users assigned: `db.userRole.count({ where: { roleId } })` -- if > 0, throw `"Роль назначена ${count} пользователям. Сначала снимите роль."`
   - `db.role.delete({ where: { id: roleId } })`
   - Audit entry with action "DELETE"
   - `revalidatePath("/settings/roles")`

6. `getPermissionsByModule()` -- Return permissions grouped by module for the matrix UI.
   - No permission check needed (used in forms)
   - `db.permission.findMany({ orderBy: [{ module: "asc" }, { name: "asc" }] })`
   - Group by module field: `Record<string, { id: string, code: string, name: string }[]>`
   - Module display names mapping:
