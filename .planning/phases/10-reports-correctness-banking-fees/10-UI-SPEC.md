## Typography

| Role    | Size | Weight         | Line Height |
| ------- | ---- | -------------- | ----------- |
| Body    | 14px | 400 (regular)  | 1.5         |
| Label   | 14px | 600 (semibold) | 1.4         |
| Heading | 20px | 600 (semibold) | 1.2         |
| Display | 30px | 600 (semibold) | 1.2         |

Notes:

- Display (30px semibold) used exclusively for StatCard large values (formatMoney output on dashboard)
- Heading (20px semibold) used for CardTitle in report sections and settings page titles
- Label (14px semibold) used for table headers, form labels, summary card titles
- Body (14px regular) used for table cells, descriptions, muted-foreground text
- 2-weight constraint: 400 (regular) + 600 (semibold) only throughout this phase

## Layout Specifications

### Dashboard — Gross vs Net Profit (FEE-05)

Current state: single StatCard "Прибыль" in row 1 (4 cards grid).

New state: replace single "Прибыль" card with TWO cards in same grid:

```
[Продаж сегодня] [Выручка сегодня] [Средний чек] [Валовая прибыль] [Чистая прибыль]
```

Grid changes from `lg:grid-cols-4` to `lg:grid-cols-5` on row 1. If 5 columns is too tight on 1280px viewport, use alternative:

```
Row 1: [Продаж] [Выручка] [Средний чек]         -- lg:grid-cols-3
Row 2: [Валовая прибыль] [Чистая прибыль] [Маржа] -- lg:grid-cols-3
```

Decision: use **two rows** layout (3+3 grid) for breathing room. This is consistent with the existing pattern where row 2 already uses `lg:grid-cols-3`.

- "Валовая прибыль" StatCard: value in `text-blue-600`, description `Маржа {X}%`
- "Чистая прибыль" StatCard: value in `text-green-600` / `text-red-600` (conditional), description `После комиссий и расходов`

### Profit Report — Extended Summary (REP-03, FEE-04, REP-06)

Current layout (2 rows of summary cards):

```
Row 1: [Выручка] [Себестоимость] [Валовая прибыль] [Маржа]  -- grid-cols-4
Row 2: [Списания] [Чистая прибыль]                           -- grid-cols-3
```

New layout (3 rows of summary cards):

```
Row 1: [Выручка] [Возвраты] [Себестоимость] [Валовая прибыль] [Маржа]  -- grid-cols-5
Row 2: [Списания] [Комиссии банка] [Trade-in расходы]                   -- grid-cols-3
Row 3: [Чистая прибыль] [Чистая маржа]                                  -- grid-cols-2
```

- "Возвраты" card: value in `text-red-500`, displays negative amount with minus sign
- "Комиссии банка" card: value in `text-orange-500`
- "Trade-in расходы" card: value in `text-orange-500`
- "Чистая прибыль" card: full-width emphasis, larger value (text-2xl font-semibold)

### Cash Report — New Tab (REP-07)

Add "Кассовый отчёт" tab to reports-page-client.tsx TabsList.

Tab order: Продажи | Прибыль | Склад | Продавцы | **Кассовый отчёт**

Layout structure:

```
[DateRangePicker] [Store selector]

-- Summary strip --
[Наличные: X] [Карта: X] [СБП: X] [Перевод: X] [Рассрочка: X]   -- grid-cols-5

-- Per-shift breakdown --
Card: "Смена #N — Кассир Имя — DD.MM.YYYY"
  Table:
    | Метод оплаты | Кол-во операций | Приход | Расход | Итого |
    | Наличные      | 12              | 50000  | 3000   | 47000 |
    | Карта          | 8              | 30000  | 0      | 30000 |
    ...
  Separator
  Сверка кассы:
    | Ожидаемый остаток | Фактический | Расхождение |
    | 47000             | 46500       | -500        |

[Repeat per shift in period]

-- Period totals --
Card: "Итого за период"
  Same table structure, aggregated
```

- Payment method names use Badge component with `variant="outline"`
- Discrepancy row: `text-red-600` if negative, `text-green-600` if zero or positive
- Shifts ordered chronologically (oldest first)

### Fee Settings Form (FEE-01)

Location: `/settings` page, new section or sub-page.

Layout:

```
Card: "Комиссии по методам оплаты"
  CardHeader: title + description "Укажите процент комиссии банка для каждого метода оплаты"

  Table:
    | Метод оплаты | Ставка, % | Пример (10 000 р) |
    | Наличные      | [0.00]    | Комиссия: 0 р     |
    | Карта          | [2.00]    | Комиссия: 204.08 р |
    | СБП            | [0.70]    | Комиссия: 70.49 р  |
    | Перевод        | [1.00]    | Комиссия: 101.01 р |
    | Рассрочка      | [3.00]    | Комиссия: 309.28 р |

  Footer: [Сохранить ставки] button, right-aligned
```

- Rate input: `<Input type="number" step="0.01" min="0" max="99" />` width 80px, right-aligned text
- Example column: computed realtime as user types, uses reverse percentage formula
- Example amount fixed at 10000 for clarity
- CASH row: input disabled, value locked at 0.00, row has `bg-muted/50` background

### POS Payment Dialog — Fee Display (FEE-03)

Extend existing `payment-dialog.tsx`.

When selected method is not CASH (or in combined mode for non-CASH portions):

```
Before:                          After:
---------------------           ---------------------
Итого: 10 000 р                 Цена товара: 10 000 р
                                Комиссия банка: 204.08 р
[Оплатить]                      Итого к оплате: 10 204.08 р
                                [Оплатить 10 204.08 р]
```

- "Комиссия банка" line: `text-orange-500 text-sm`
- "Итого к оплате" line: `text-lg font-semibold`
- Fee line appears/disappears with CSS transition (animate-in when method changes)
- For CASH: fee line hidden, show only "Итого: X р" (current behavior)
- For combined payment: show fee breakdown per non-CASH method, then total fee

## Interaction Contracts

### Fee Settings Form

| Interaction              | Behavior                                                                      |
| ------------------------ | ----------------------------------------------------------------------------- |
| Rate input change        | Recompute example column in realtime (no server call)                         |
| Save button click        | POST to server, show loading spinner on button, disable inputs during save    |
| Save success             | Sonner toast "Ставки комиссий сохранены", button returns to idle              |
| Save error               | Sonner toast with error message, button returns to idle, inputs retain values |
| CASH row input           | Disabled, value locked at 0.00                                                |
| Invalid rate (>99 or <0) | HTML validation prevents, no custom error UI needed                           |

### POS Fee Display

| Interaction               | Behavior                                                             |
| ------------------------- | -------------------------------------------------------------------- |
| Method switch to non-CASH | Fee line appears with fade-in (150ms), total recalculated            |
| Method switch to CASH     | Fee line disappears, total reverts to base price                     |
| Combined payment toggle   | Per-method fee lines appear for each non-CASH method with amount > 0 |
| Combined amount change    | Fee recalculated per method in realtime                              |

### Cash Report

| Interaction         | Behavior                                                    |
| ------------------- | ----------------------------------------------------------- |
| Date range change   | Trigger server fetch with loading skeleton                  |
| Store change        | Trigger server fetch with loading skeleton                  |
| No shifts in period | Show empty state (heading + body from copywriting contract) |

### Profit Report (extended)

| Interaction         | Behavior                                                 |
| ------------------- | -------------------------------------------------------- |
| Date range change   | Same as current -- useTransition fetch with skeleton     |
| Zero revenue period | Show "Нет данных за выбранный период" (existing pattern) |
